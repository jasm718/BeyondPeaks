# BeyondPeaks 阶段一：首页 3D 地球选山

## 需求背景
BeyondPeaks 的完整产品目标是通过用户真实爬升数据推动虚拟登山进度。阶段一不实现完整攀爬闭环，只验证首页核心体验：用户打开 App 后，能通过真实 3D 地球直观看到世界名山位置，并点选山峰查看基础信息。当前仓库只有产品说明和设计规范，没有可运行框架，因此需要先定义一个最小、清晰、可扩展的 Flutter iOS 骨架，先把“选山”这条主链路做稳。

## 解决方法
使用 Flutter 构建阶段一 iOS App，`flutter_riverpod` 管理状态，`go_router` 管理底部导航。首页加载本地 `earth.glb`，展示真实 3D 地球，支持自动慢转、旋转、缩放和点选。首版固定展示 3 座山峰：珠穆朗玛峰、乔戈里峰、马特洪峰。山峰数据使用经纬度映射到地球表面；3D 场景只显示点标记，名称和海拔用 Flutter 层叠加。用户点选山峰后，在首页底部弹出抽屉式详情浮层，显示基础信息和路线概览，不进入独立详情页，不放操作按钮。底部导航固定为 `首页 / 攀爬 / 收藏 / 设置`，其中除首页外先做占位页。所有数据来自本地静态资源，当前选中山峰只在当前 App 会话中跨页面保留，不跨重启持久化。

## 范围
做：
- Flutter iOS 单机 App 框架
- 首页真实 3D 地球
- 地球旋转、缩放、点选、自动慢转
- 3 座固定山峰的本地静态数据：珠穆朗玛峰、乔戈里峰、马特洪峰
- 每座山 1 条默认路线摘要
- 山峰标记显示点、山名、海拔
- 首页底部抽屉详情浮层
- 选中状态在当前 App 会话内跨页面保留
- 攀爬、收藏、设置占位页

不做：
- 攀爬闭环
- 用户账号
- 后端接口
- 云同步
- 远程山峰数据加载
- 独立山峰详情页
- 收藏和统计真实逻辑
- 统计页入口
- 故事系统
- 多路线切换
- 选中状态跨重启持久化

## 用户故事
1. 作为用户，我想打开 App 后看到真实 3D 地球，以便直观看到山峰所在位置。
2. 作为用户，我想旋转和缩放地球，以便浏览不同地区的山峰。
3. 作为用户，我想看到山峰名称和海拔标记，以便快速识别目标山峰。
4. 作为用户，我想点选山峰后看到底部详情浮层，以便了解基础信息和路线概览。
5. 作为用户，我想切换到底部其他入口后再返回首页仍保留当前选中山峰，以便继续浏览上下文。
6. 作为用户，我想看到攀爬、收藏、设置入口的占位页，以便理解应用后续结构。
7. 作为用户，我想在 3D 地球或本地数据加载失败时看到明确失败态，以便知道当前页面不可用。
8. 作为用户，我首次进入首页时不希望默认弹出山峰详情，以便先自由浏览地球。

## 实现方案

### 技术约束
- Flutter iOS 单机 App。
- 状态管理使用 `flutter_riverpod`。
- 路由使用 `go_router` 的 `ShellRoute` 承载底部导航。
- 3D 技术使用 `three_js`。
- 地球资源使用本地 `assets/models/earth.glb`。
- 山峰数据完全本地静态，不接后端，不做云同步。
- 山峰使用经纬度计算 3D 坐标。
- `earth.glb` 必须以球心为原点，使用标准球体半径和固定朝向；经纬度到 3D 坐标的映射依赖该模型约束。
- 山峰名称和海拔由 Flutter 层叠加展示。
- 山峰标签固定屏幕大小，不随地球缩放。
- 地球背面的山峰标记和标签不显示。
- 选中山峰状态在当前 App 会话内跨页面保留，不跨重启持久化。
- 视觉样式遵循 `DESIGN.md` 的 Stone & Slate 设计系统。

### API接口

首版无远程接口，只有本地数据和状态接口。

| 接口名称 | 类型 | 说明 |
| --- | --- | --- |
| 山峰数据读取 | 本地静态数据 | 返回固定 3 座山峰及路线摘要 |
| 当前选中山峰 | Riverpod Provider | 保存当前选中山峰，当前 App 会话内跨页面保留 |
| 地球点选事件 | 组件回调 | 命中山峰后更新选中状态 |
| 路由切换 | `go_router` ShellRoute | 管理首页、攀爬、收藏、设置入口 |

### 数据规则

| 数据 | 规则 |
| --- | --- |
| 首版山峰 | 固定为珠穆朗玛峰、乔戈里峰、马特洪峰 |
| 山峰字段 | 名称、区域、海拔、纬度、经度、默认路线名称、路线摘要 |
| 路线数量 | 每座山只保留 1 条默认路线摘要 |
| 纬度 | 必填，范围 `-90..90` |
| 经度 | 必填，范围 `-180..180` |
| 非法经纬度 | 直接抛错，不进入渲染 |

### 交互规则

| 交互 | 规则 |
| --- | --- |
| 首次进入首页 | 不默认选中山峰，不显示详情浮层 |
| 自动慢转 | 首次进入首页自动慢转；用户拖动后停止，不自动恢复 |
| 重新进入首页 | 重新启用自动慢转；如果当前会话存在选中山峰，则重新显示浮层 |
| 缩放 | 最小 `0.8x`，最大 `2.0x` |
| 点按判定 | 手指移动小于 `8px` 视为点按，否则视为拖动 |
| 标记可见性 | 只显示地球正面的山峰标记和标签 |
| 标签缩放 | 标签固定屏幕大小，不随地球缩放 |
| 详情浮层字段 | 名称、区域、海拔、默认路线名称、路线摘要 |
| 详情浮层关闭 | 点空白地球区域关闭；切换页面关闭；回首页按当前会话选中状态重新显示 |

### 异常处理

| 场景 | 处理方式 | 用户提示 |
| --- | --- | --- |
| `earth.glb` 缺失或加载失败 | 立即中止首页 3D 区域渲染，底部导航仍可用 | 地球模型加载失败 |
| 山峰经纬度非法 | 开发期直接抛错，阻止错误数据进入渲染 | 不展示用户提示 |
| 山峰数据为空 | 首页显示空状态，不渲染标记 | 暂无可展示山峰 |
| 3D 初始化失败 | 首页 3D 区域显示失败态，不进入交互逻辑，底部导航仍可用 | 3D 地球初始化失败 |
| 点选命中失败 | 不改变当前选中山峰 | 无提示 |
